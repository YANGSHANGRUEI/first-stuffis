# 繳交前 Demo 驗收清單

執行：`streamlit run app.py`

每步預期結果打勾後，截圖可貼進期末報告。

## 1. 首頁（未登入）

- [ ] 左側只有 **登入** 與 **註冊**（無首頁、無功能頁）
- [ ] 直接開上傳／瀏覽 URL 會被 `st.stop()` 擋住

## 2. 註冊

- [ ] 刪除 `data/users.json` 後仍可註冊（自動建檔）
- [ ] 空白帳密、重複帳號會被警告
- [ ] 新帳號 `tokens: 0`、`unlocked: []`

## 3. 登入／登出

- [ ] 錯誤帳密顯示「帳號或密碼錯誤」（不區分）
- [ ] 登入後左側出現 **首頁** 與 **功能**（上傳、瀏覽、個人）；**登入／註冊消失**
- [ ] 首頁按 **登出** → 回到僅首頁＋帳號
- [ ] 重新整理仍維持登入（`current_session.json` + `sig`）

## 4. 上傳

- [ ] 四層下拉（法領域→科目→老師→年份）來自 `config/taxonomy.json`
- [ ] 上傳成功 +3 代幣
- [ ] 相同四層＋相同作答重送 → 防重複警告

## 5. 瀏覽／解鎖

- [ ] 四層篩選可縮小列表
- [ ] `legacy_anonymous` 顯示「不開放」、無解鎖鈕
- [ ] 代幣足夠可解鎖；不足顯示錯誤
- [ ] 已解鎖內容再次進入不重扣代幣

## 6. 個人頁

- [ ] 三欄 metric：代幣、上傳筆數、已解鎖數
- [ ] **已解鎖清單** 顯示分類標籤（如 `刑法｜總論｜林教授｜113`）

## 7. 隱私（報告用，不需截圖）

- [ ] `answers.csv` 無明文帳號，只有 `uploader_id`
- [ ] `current_session.json` 只有 `user_id` + `sig`

全部打勾 → 可作為「整合測試通過」的繳交依據。
